### Что это

Статический сайт для GitHub Pages, который показывает **только одно число** — текущее место **Сагдуллин Марсель** среди **11 класс** на `reg.algocode.ru`.

Важно: браузер с GitHub Pages **не может** напрямую скачать `/_api/reg_2026.json` из-за CORS, поэтому число обновляет **GitHub Actions**, записывая результат в `place.json`.

### Как опубликовать на GitHub Pages

- **Шаг 1**: создай репозиторий на GitHub и залей туда файлы из этой папки
- **Шаг 2**: в репозитории открой **Settings → Pages**
  - **Source**: `Deploy from a branch`
  - **Branch**: `main` и папка `/ (root)`
- **Шаг 3**: открой вкладку **Actions** и запусти workflow **Update place.json** (кнопка “Run workflow”)
  - дальше он будет запускаться каждые ~5 минут сам

### Что где лежит

- `index.html`: страница с одним большим числом
- `place.json`: файл с текущим местом (его обновляет Actions)
- `compute_place.py`: скрипт, который скачивает standings и считает место
- `.github/workflows/update-place.yml`: расписание обновления
